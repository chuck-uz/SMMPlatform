# Vault проекта (база контекста)

Это хранилище Obsidian — единая память проекта. Claude читает и пишет сюда контекст по ходу разработки.

> **Правило доступа:** всё общение с этим vault — **только через `obsidian-mcp-connector`**, не через прямое редактирование файлов на диске. Подробности — в `CLAUDE.md` в корне репозитория.

## Структура
- `MASTER-CONTEXT.md` — единый источник правды: концепция, решения, ограничения, модули, смета, дорожная карта. Сюда положить наш мастер-файл.
- `roadmap.json` — задачи и статусы (единый источник для разработки и для сайта-дорожной карты). Обновляется по ходу работы.
- `context/` — рабочие заметки по модулям (что известно, как устроено, открытые вопросы). Claude читает их перед задачей. Шаблон — `context/_template.md`.
- `decisions/` — лог решений: что и почему сделали (архитектура, выбор подхода). Claude дописывает после задачи. Шаблон — `decisions/_template.md`.

## Петля работы
1. Перед задачей — прочитать нужный раздел `MASTER-CONTEXT.md` и профильную заметку из `context/`.
2. Сделать задачу по критерию `done_when` из `roadmap.json`.
3. После — дописать запись в `decisions/<модуль>.md` и обновить `status` задачи в `roadmap.json`.

## Как обновлять статус задачи
Статусы: `todo` → `in_progress` → `done` (или `blocked`). Обновление = смена поля `status` (и при желании `note`, `updatedAt`) у нужного `id` в `roadmap.json`. Через коннектор: прочитать файл → изменить → перезаписать.

## Заметки
Веди заметки **чистым markdown с ясными заголовками** — их читает и человек в Obsidian (граф, связи), и Claude как текст. Вики-ссылки `[[...]]` и dataview — для тебя; для Claude важна ясная структура заголовков.
