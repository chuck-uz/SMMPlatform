# App Review

> **Источник:** https://developers.facebook.com/docs/app-review
> **Снято:** 2026-07-17 · **Треки роадмапа:** BV6, DM3/DM5, CT6, AD8
> Снимок актуальных доков Meta. Страница `/app-review/requirements` вернула 404 — детали чек-листа сверять в дашборде при подаче.

## Когда нужен App Review

- Нужен, если приложением будут пользоваться люди **без роли в приложении** или роли в связанном Business.
- **Исключение:** если только пользователи с ролью — App Review не требуется (но тогда это не продакшн для клиентов).

## Логика доступа

- Одобренные permissions/features можно запрашивать у **всех** пользователей.
- Неодобренные — только у пользователей с ролью.

## Что тестирует Meta

- Функциональная проверка реального использования каждого запрошенного permission/feature.
- **Если приложение недоступно для теста — вся заявка отклоняется.**
- Если конкретный permission нельзя протестировать — частичный отказ по нему.

## Что готовим к подаче (практика)

- **Live Mode** приложения.
- Валидный **Privacy Policy URL** (у нас `/privacy`, `/terms`, `/data-deletion` — сделано в DM4-подготовке).
- **Screencast** по каждому разрешению (реальный сценарий использования).
- **Пошаговые инструкции воспроизведения** + тестовые доступы для ревьюера (черновики текстов уже готовились).
- Иконка приложения (1024×1024 — сделано).
- Для многих permissions в Advanced — предварительно **Business Verification** (см. `11-business-verification.md`).

## Связь с задачами

- CM1/CM2 → сценарий «Управление комментариями» (BV6).
- DM → DM3 (подготовка соответствия) + DM5 (подача, скринкасты, Live+webhook).
- CT → CT6 (ревью на публикации).
- AD → AD8 (ревью рекламы + Business Verification).
